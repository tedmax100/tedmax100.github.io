Go Module基礎

出現原因

  1. GOPATH不符合一般開發者的習慣; 大家習慣用maven, node module之類的方式.
  2. GOPATH無法有效管理板依賴, 沒有辦法表明所依賴的包的版本.

環境準備

  • Go version >= 1.11
  • GO111MODULE=on

GoMod effect immport package

  • 可以在$GOPATH之外的地方建立專案
  • 該專案Go Module開啟後, 下載的package會放在$GOPATH/pkg/mod下.
  • $GOPATH/bin的功能依然保持

Go Mod Commands


有兩種方式能定義一個正確的Go module

就會在該專案下生出了go.mod文件了.

Syntax of go.mod

  • module
    • 定義模組路徑
  • go
    • 定義go version
  • require
    • 指定依賴的功能包和其版本或是[預設是最新版]
  • exclude
    • 忽略該功能包和其版本
  • replace
    • 替換依賴的功能包
